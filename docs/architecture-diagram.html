🌊 End-to-End Flow: "Generate Test Cases"

1

👤 User clicks "Generate"

Frontend collects inputs (text + Figma URL + screenshots…) and POSTs to /api/generate

2

🔌 Router validates request

Looks up project, feature, and existing test cases for context

3

🔀 Parser dispatch (parallel)

Each input runs through its parser → merged into a single ParsedInput

4

🤖 LLM call

Build prompt → send to OpenAI/Anthropic/Gemini → receive JSON test cases

5

🧹 Dedup & post-process

Check against existing cases → assign IDs (TC_001…) → attach source_ref

6

💾 Persist & respond

Save to SQLite → log to input_history → return JSON → UI updates

🗄️ Database Schema

👥 users
  • 🔑 id (PK)
  • name
  • email (unique)
  • password_hash
  • created_at
📂 projects
  • 🔑 id (PK)
  • ↗ user_id (FK)
  • name, description
  • context (JSON)
  • created_at, updated_at
⭐ features
  • 🔑 id (PK)
  • ↗ project_id (FK)
  • name
  • description
  • sort_order
✅ test_cases
  • 🔑 id (PK)
  • ↗ project_id (FK)
  • ↗ feature_id (FK)
  • title, type, priority
  • preconditions, steps
  • expected_result, source_ref
📊 generations
  • 🔑 id (PK)
  • ↗ project_id (FK)
  • model, tokens
  • timestamp
🔐 app_settings
  • API keys (JSON blob)
  • UI overrides
  • per-user settings
⚡ figma_cache
  • cache_key
  • file_key, node_id
  • parsed_json
  • cached_at
🌐 browser_sessions
  • recorded user flows
  • step-by-step actions
  • screenshots per step
